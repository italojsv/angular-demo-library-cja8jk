Bypass HTML Sanitizing

Hello !

This demo will present you with a custom greeting and demonstrate the danger with bypassing Angular sanitization on untrusted inputs. The user's name may be passed to the application via the 'name' attribute in the query string or the 'trusted_name' attribute in the query string (e.g. /ElementRef?trusted_name=Bob). Sanitization will be enforced if using the 'name' attribute, and it will be disabled if using the 'trusted_name' attribute.

Tip

Angular automatically sanitizes data placed into the template unless sanitization is explicitly disabled with the sanitizer's 'bypassSecurityTrust*' functions. HTML sanitization has been disabled for the 'trusted_name' parameter, demonstrating how the application becomes vulnerable to XSS attacks given a malicious input such as /?trusted_name=<img src=x onerror=alert("XSS_SUCCESS!")>.
